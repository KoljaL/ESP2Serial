# ESP8266 Target Starter

Generated by Copilot

Dieses Teilprojekt ist der Startpunkt fuer den Sonoff iFan04 auf ESP8266-Basis.
Es enthaelt:

- WLAN-Connect mit Werten aus `.env`
- automatischen Fallback auf einen eigenen Access Point
- Arduino OTA fuer weitere Uploads nach dem ersten Flash
- Weboberflaeche fuer Status, Konfiguration und empfangene Fernbedienungssignale
- persistente Konfiguration im Flash, die `.env`-Defaults zur Laufzeit uebersteuern kann

## Warum ein separates Projekt?

Das Root-Projekt in diesem Workspace ist die ESP32-Bridge fuer den ersten seriellen Flash.
Dieses Teilprojekt ist die eigentliche Ziel-Firmware fuer den ESP8266.

## Board-Wahl

PlatformIO hat kein eigenes `ifan04`-Boardprofil. Dieses Projekt verwendet daher `esp8285`, weil der iFan04 typischerweise auf einem ESP8285-basierten Sonoff-Design aufsetzt.

## Dateien

- `platformio.ini`: zwei Umgebungen, einmal fuer den ersten seriellen Upload und einmal fuer OTA
- `.env.example`: Vorlage fuer lokale WLAN- und OTA-Werte
- `extra_scripts/load_env.py`: liest `.env` und uebergibt Werte an den Build
- `src/main.cpp`: WLAN-First, AP-Fallback, OTA, Web UI, persistente Konfiguration und Signalansicht

## Einrichtung

1. In diesem Ordner `.env` bearbeiten.
2. Diese Werte setzen:
   - `WIFI_SSID`
   - `WIFI_PASSWORD`
   - `OTA_HOSTNAME`
   - optional `FALLBACK_AP_SSID`
   - optional `FALLBACK_AP_PASSWORD`
3. Hinweis: Wenn du spaeter Werte ueber die Weboberflaeche speicherst, liegen diese im Flash auf dem Geraet und haben Vorrang vor den `.env`-Werten, bis du sie ueber die Weboberflaeche wieder loeschst.

## Erster Flash ueber die ESP32-Bridge

Der erste Flash bleibt seriell. Dazu den ESP8266 manuell in den Bootloader bringen und dann aus diesem Ordner bauen und hochladen.

```bash
platformio run -e ifan04_serial -t upload --upload-port /dev/cu.SLAB_USBtoUART
```

Den Upload-Port musst du auf den USB-Port deines ESP32-Bridge-Boards setzen.

## IP-Adresse nach erfolgreichem WLAN-Connect

Die Firmware gibt die vergebene IP direkt ueber die serielle Konsole aus.

Die Firmware selbst laeuft jetzt mit 9600 Baud, weil die Fernbedienungssignale ueber UART RX auf GPIO3 mit 9600 ankommen. Fuer das Monitoring also besser:

```bash
platformio device monitor -b 9600 --port /dev/cu.SLAB_USBtoUART
```

Zusatzlich versucht OTA standardmaessig den Hostnamen aus `OTA_HOSTNAME` zu verwenden, also zum Beispiel `ifan04.local`.

## OTA-Upload mit PlatformIO

Wenn `.local` in deinem Netz funktioniert:

```bash
platformio run -e ifan04_ota -t upload
```

Wenn du lieber direkt per IP hochladen willst:

```bash
ESP8266_OTA_TARGET=192.168.1.123 platformio run -e ifan04_ota -t upload
```

## Fallback-AP

Wenn kein Connect ins vorhandene WLAN gelingt, startet der ESP8266 einen eigenen Access Point.

- SSID: Wert aus `FALLBACK_AP_SSID` plus Chip-ID
- IP: typischerweise `192.168.4.1`
- Passwort: nur aktiv, wenn `FALLBACK_AP_PASSWORD` mindestens 8 Zeichen lang ist

## Weboberflaeche

Die Weboberflaeche ist sowohl im normalen WLAN-Betrieb als auch im Fallback-AP verfuegbar.

- im WLAN: `http://<ip-aus-serieller-ausgabe>/` oder `http://<OTA_HOSTNAME>.local/`
- im Fallback-AP: typischerweise `http://192.168.4.1/`

Die Seite zeigt:

- aktuellen Netzwerkmodus und die aktive IP
- gespeicherte WLAN- und OTA-Konfiguration
- zuletzt empfangene Fernbedienungsframes als HEX und als lesbare Textdarstellung

## Fernbedienungssignale

Die Anwendungslogik lauscht auf eingehende UART-Daten der Fernbedienung hierauf:

- RX-Pin: `GPIO03`
- Baudrate: `9600`

Eingehende Bytes werden zu Frames zusammengefasst und auf der Weboberflaeche angezeigt. Ein Frame endet bei Zeilenende oder nach einer kurzen Empfangspause.
